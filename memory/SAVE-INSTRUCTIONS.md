## 🎯 Обзор системы

Система памяти Maribulka поддерживает **три режима сохранения**:

| Режим | Когда | Что делает |
|-------|-------|------------|
| **Автосохранение при выходе** | При завершении сессии | Краткое резюме → `auto-save.md` + `changelog.md` |
| **Ручное сохранение** | По команде `/сохранить` | Полное резюме с категоризацией → выбранный файл |
| **Промежуточное сохранение** | Каждые 10 минут | Прогресс → `changelog.md` |

## 🚀 Ручное сохранение (команды)

### Базовые команды

```
/сохранить              → Сохранить текущий прогресс в changelog.md
/сохранить в patterns   → Сохранить как новый паттерн в patterns.md
/сохранить в traps      → Сохранить как новый баг/ловушку в traps.md
/сохранить в styles     → Сохранить изменения стилей в styles.md
/сохранить в architecture → Сохранить изменения архитектуры в architecture.md
/сохранить в business   → Сохранить бизнес-процесс в business-process.md
```

### Примеры использования

**После завершения задачи:**
```
Пользователь: Мы закончили рефакторинг кнопок. Все кнопки теперь используют btnGlass.
Пользователь: /сохранить в patterns
```

**После исправления бага:**
```
Пользователь: Исправил баг с toISOString() — теперь используем кастомную функцию formatDate.
Пользователь: /сохранить в traps
```

**После изменения архитектуры:**
```
Пользователь: Перенёс медиа-файлы вне dist/, создал симлинк на сервере.
Пользователь: /сохранить в architecture
```

## 🔄 Автосохранение при выходе

### Как работает

1. При завершении сессии (когда вы закрываете терминал/Qwen Code)
2. Хук `session-end` автоматически запускается
3. Создаётся краткое резюме:
   - Какие файлы были изменены
   - Краткое описание изменений
4. Запись добавляется в:
   - `memory/auto-save.md` (полная история сессий)
   - `memory/changelog.md` (история изменений проекта)

### Формат записи

```markdown
### 📅 2026-04-02 15:30

**Изменённые файлы:**
- src/components/TopBar.vue
- src/assets/buttonGlass.css

**Что сделано:**
Рефакторинг кнопок TopBar: миграция на btnGlass, добавлены модификаторы bigIcon и iconText.
```

## ⏰ Промежуточное автосохранение

### Настройка таймера

Для включения автосохранения каждые 10 минут, добавьте в начало сессии:

```
Пользователь: Включи автосохранение каждые 10 минут
```

Агент создаст фоновый процесс, который будет:
- Отслеживать изменённые файлы
- Каждые 10 минут записывать прогресс в `changelog.md`
- Останавливаться при завершении сессии

## 🎯 Лучшие практики

### ✅ Что сохранять вручную

| Тип информации | Куда сохранять |
|---------------|----------------|
| Новый паттерн кода | `patterns.md` |
| Исправленный баг | `traps.md` |
| Изменение CSS | `styles.md` |
| Новая архитектура | `architecture.md` |
| Новое правило | `MEMORY.md` |
| Бизнес-логика | `business-process.md` |

### ✅ Когда использовать команды

- **После завершения крупной задачи** → `/сохранить в <категория>`
- **Перед перерывом** → `/сохранить` (в changelog)
- **При обнаружении бага** → `/сохранить в traps`
- **После рефакторинга** → `/сохранить в patterns` или `styles`

### ⚠️ Важные заметки

1. **Автосохранение при выходе** работает только если сессия завершается нормально
2. **Ручное сохранение** требует явной команды
3. **Промежуточное автосохранение** нужно включать явно в начале сессии

## 🔧 Технические детали

### Хук SessionEnd

- **Путь:** `.qwen/extensions/superpowers/hooks/session-end`
- **Триггер:** Завершение сессии Qwen Code
- **Действие:** Запись в `auto-save.md` и `changelog.md`

### Переменные окружения

Хук использует:
- `QWEN_SESSION_SUMMARY` — резюме сессии (предоставляется Qwen Code)
- `PROJECT_ROOT` — корень проекта (`D:/GitHub/maribulka`)
