# 📖 Инструкция по использованию MCP памяти

## 🎯 Общее описание

Эта инструкция описывает, как эффективно использовать MCP серверы памяти для работы с проектом Maribulka. Все общение ведется **исключительно на русском языке**.

## 🔄 Жизненный цикл работы с MCP памятью

### НАЧАЛО НОВОГО ЧАТА

**Шаг 1: Восстановление контекста**
```
// Получить список всех топиков контекста
context list

// Получить текущую рабочую задачу
context get topic="current-task"

// Получить общий статус проекта
context get topic="project-status"
```

**Шаг 2: Проверка сохраненной памяти**
```
// Список всех сохраненных записей
memory list

// Поиск ключевой информации
memory search query="Maribulka"
memory search query="текущие задачи"
memory search query="календарь"

// Получение критических правил
memory retrieve key="communication-rule"
memory retrieve key="critical-rules"
memory retrieve key="project-structure"
```

### В ПРОЦЕССЕ РАБОТЫ

**Сохранение промежуточных результатов:**
```
// Сохранить прогресс текущей задачи
context add topic="current-development" content="Работаем над мобильной адаптацией календаря"

// Сохранить важные решения
memory store key="calendar-mobile-decision" value="Используем flexbox для адаптивной сетки" ttl=86400

// Записать промежуточные результаты
memory store key="progress-2026-02-18" value="Завершена реализация базовой верстки, начинаем JS функциональность" ttl=43200
```

### В КОНЦЕ СЕССИИ

**Сохранение итогов:**
```
// Сохранить финальный контекст
context add topic="session-summary" content="Завершена мобильная адаптация календаря, все тесты пройдены"

// Записать результаты работы
memory store key="session-results-2026-02-18" value="Мобильная версия календаря готова к продакшену" ttl=0

// Обновить статус задач
memory store key="current-tasks" value="✅ Мобильная адаптация календаря, ⏳ Тестирование на реальных устройствах" ttl=0
```

## ⚡ Быстрые команды

### Для начала сессии:
```
context list
memory search query="текущие задачи"
memory retrieve key="communication-rule"
```

### Для проверки статуса:
```
context get topic="current-task"  
memory retrieve key="current-tasks"
memory retrieve key="project-status"
```

### Для сохранения прогресса:
```
context add topic="progress" content="Описание текущей работы"
memory store key="today-progress" value="Что было сделано" ttl=86400
```

## 🔧 Полезные советы

1. **Всегда начинайте с проверки контекста** - это экономит время
2. **Регулярно сохраняйте промежуточные результаты** - предотвращает потерю прогресса  
3. **Используйте осмысленные ключи** - облегчает поиск информации
4. **Устанавливайте разумные TTL** - важная информация без истечения срока
5. **Комбинируйте инструменты** - context для текущего контекста, memory для долгосрочного хранения

## 📌 Правильный синтаксис вызова MCP инструментов

### Как использовать команды на практике:
В реальной работе с MCP серверами команды должны быть оформлены как вызовы соответствующих инструментов. Ниже приведены примеры правильного синтаксиса:

**Для получения списка контекстов:**
```
context({ action: "list" })
```

**Для получения конкретного контекста:**
```
context({ action: "get", topic: "current-task" })
```

**Для добавления контекста:**
```
context({ action: "add", topic: "mobile-dev", content: "Работаем над адаптацией интерфейса" })
```

**Для получения списка записей в памяти:**
```
memory({ action: "list" })
```

**Для поиска в памяти:**
```
memory({ action: "search", query: "календарь" })
```

**Для получения конкретной записи из памяти:**
```
memory({ action: "retrieve", key: "project-structure" })
```

**Для сохранения в память:**
```
memory({ action: "store", key: "decision-2026-02-18", value: "Реализуем адаптивный календарь", ttl: 86400 })
```
