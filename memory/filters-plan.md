## 🔧 Технические требования

### 1. Компонент универсального фильтра

**Создать:** `components/ui/FilterBuilder.vue`

**Функционал:**
- Конструктор фильтров (добавить/удалить условие)
- Типы условий: равно, содержит, больше, меньше, диапазон
- Сохранение пресетов в localStorage
- Экспорт/импорт пресетов (JSON)
- Сброс всех фильтров

**Пример использования:**
```vue
<FilterBuilder
  :fields="filterFields"
  v-model="activeFilters"
  @apply="applyFilters"
  @reset="resetFilters"
/>
```

---

### 2. Компонент поиска с автодополнением

**Создать:** `components/ui/SearchInput.vue`

**Функционал:**
- Дебаунс ввода (300мс)
- Автодополнение из БД
- Подсветка совпадений
- История поисковых запросов
- Быстрые теги (последние запросы)

---

### 3. Компонент диапазона дат

**Создать:** `components/ui/DateRangePicker.vue`

**Функционал:**
- Выбор периода (с/по)
- Быстрые пресеты (сегодня, неделя, месяц, квартал, год)
- Кастомный диапазон
- Валидация (дата начала ≤ дата окончания)

---

**Функционал:**

  const loadFilters = () => {...}

  // Сброс фильтров
  const resetFilters = () => {...}
  
  // Применение фильтров к данным

  return {

## 🎨 UI/UX требования

### 1. Панель фильтров

**Расположение:** Над таблицей, под заголовком

**Состояния:**
- **Свёрнута** - только активные фильтры-теги + кнопка "Фильтры"
- **Развёрнута** - все поля фильтров

**Поведение:**
- Автоматическое сворачивание после применения
- Сохранение состояния (развёрнута/свёрнута) в localStorage
- Анимация раскрытия (300мс)

### 2. Активные фильтры

**Отображение:**
- Теги с названием поля и значением
- Кнопка "×" для удаления фильтра
- Кнопка "Сбросить все"
- Счётчик активных фильтров

**Пример:**
```
[Статус: Выдано ×] [Дата: 01.02.2026 - 28.02.2026 ×] [Сбросить все]
```

### 3. Мобильная адаптация

**Брейкпоинт:** ≤768px (единый стандарт из styles.md)

**Изменения:**
- Фильтры в модалке/выезжающей панели
- Вертикальное расположение полей
- Упрощённый интерфейс (только основные фильтры)
- Кнопка "Применить" фиксируется внизу экрана
