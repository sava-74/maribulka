# 🚀 Автоматический запуск MCP серверов

## 📋 Описание

Этот набор скриптов позволяет автоматически запустить все MCP серверы проекта Maribulka в отдельных консолях/терминалах с последующей проверкой их состояния.

## 📁 Содержимое

### PowerShell скрипт (Windows)
- **`start-all-mcp-servers.ps1`** - основной скрипт для Windows

### Bash скрипт (Linux/macOS)
- **`start-all-mcp-servers.sh`** - основной скрипт для Linux/macOS

### Вспомогательные файлы
- **`check-mcp-servers.js`** - скрипт проверки состояния серверов (уже существует)

## ▶️ Использование

### Windows
```powershell
# Запуск в PowerShell
.\start-all-mcp-servers.ps1

# Или двойным кликом по файлу
start-all-mcp-servers.ps1
```

### Linux/macOS
```bash
# Сделать скрипт исполняемым (если еще не сделано)
chmod +x start-all-mcp-servers.sh

# Запуск скрипта
./start-all-mcp-servers.sh
```

## 🎯 Что делает скрипт

1. **Проверяет наличие необходимых инструментов:**
   - Node.js
   - uvx (устанавливается при необходимости)

2. **Запускает все MCP серверы в отдельных консолях:**
   - **Basic-memory сервер** (`uvx basic-memory mcp`)
   - **Sequential-thinking сервер** (`npx -y @modelcontextprotocol/server-sequential-thinking`)
   - **Context7-mcp сервер** (`npx -y @upstash/context7-mcp@latest`)
   - **Playwright-mcp сервер** (`npx @playwright/mcp@latest`)

3. **Выполняет проверку состояния серверов:**
   - Запускает `check-mcp-servers.js`
   - Показывает список активных процессов

## 🔧 Требования

### Общие требования
- Node.js версии 18 или выше
- Доступ в интернет для установки пакетов

### Windows
- PowerShell 5.0 или выше
- Возможность запуска скриптов PowerShell

### Linux/macOS
- Bash
- Совместимый терминал (GNOME Terminal, Konsole, XTerm, Tilix)

## ⚠️ Особенности

### Windows
- Каждый сервер запускается в отдельном окне PowerShell
- Окна остаются открытыми после запуска серверов
- Для закрытия серверов нужно закрыть соответствующие окна PowerShell

### Linux/macOS
- Серверы запускаются в отдельных вкладках/окнах терминала
- Поддерживаются популярные терминалы: GNOME Terminal, Konsole, XTerm, Tilix
- Если совместимый терминал не найден, серверы запускаются в фоне

## 📊 Порядок запуска серверов

1. **Basic-memory сервер** (приоритет высокий) - основная память
2. **Sequential-thinking сервер** - логический анализ
3. **Context7-mcp сервер** - управление контекстом
4. **Playwright-mcp сервер** - автоматизация браузера

## 🔍 Проверка состояния

После запуска скрипт автоматически выполняет проверку состояния серверов. Также можно вручную проверить состояние:

```bash
node check-mcp-servers.js
```

## 🛑 Остановка серверов

### Windows
Закройте окна PowerShell с запущенными серверами

### Linux/macOS
Закройте вкладки/окна терминалов или используйте:
```bash
pkill -f "modelcontextprotocol|context7-mcp|playwright/mcp|basic-memory"
```

## 📝 Примечания

- Первый запуск может занять больше времени из-за установки зависимостей
- Серверы остаются активными до закрытия соответствующих окон/терминалов
- Для постоянной работы рекомендуется использовать системные службы (systemd, Windows Services)

## 🆘 Решение проблем

### "Node.js не найден"
Установите Node.js с официального сайта: https://nodejs.org/

### "Скрипт не может быть выполнен" (Windows)
Разрешите выполнение скриптов PowerShell:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Серверы не запускаются
1. Проверьте подключение к интернету
2. Убедитесь, что порты не заняты другими приложениями
3. Попробуйте запустить серверы вручную по одному
