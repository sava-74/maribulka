## 🚀 Как использовать

### Автосохранение при выходе (работает автоматически)

Просто работайте в сессии как обычно. Когда завершите работу:
1. Закройте терминал или Qwen Code
2. Хук `session-end` автоматически запустится
3. Резюме сессии сохранится в `auto-save.md` и `changelog.md`

**Пример записи:**
```markdown
### 📅 2026-04-02 15:30

**Изменённые файлы:**
- src/components/TopBar.vue
- src/assets/buttonGlass.css

**Что сделано:**
Рефакторинг кнопок TopBar: миграция на btnGlass.
```

---

### Ручное сохранение (команды)

Используйте команды в чате для сохранения с категоризацией:

| Команда | Куда сохраняет |
|---------|---------------|
| `/сохранить` | `changelog.md` |
| `/сохранить в patterns` | `patterns.md` |
| `/сохранить в traps` | `traps.md` |
| `/сохранить в styles` | `styles.md` |
| `/сохранить в architecture` | `architecture.md` |
| `/сохранить в business` | `business-process.md` |

**Пример использования:**
```
Пользователь: Мы закончили рефакторинг кнопок. Все кнопки теперь используют btnGlass.
Пользователь: /сохранить в patterns
```

---

### Промежуточное автосохранение (каждые 10 минут)

Для включения скажите агенту в начале сессии:
```
Включи автосохранение каждые 10 минут
```

Агент будет автоматически сохранять прогресс в `changelog.md`.

## 🎯 Лучшие практики

### ✅ Что делать

1. **Перед перерывом:** `/сохранить` — сохранит текущий прогресс
2. **После задачи:** `/сохранить в <категория>` — сохранит знание
3. **После бага:** `/сохранить в traps` — сохранит описание бага
4. **Просто закройте терминал** — автосохранение сработает

### ⚠️ Важные заметки

1. **Автосохранение при выходе** работает только при нормальном завершении сессии
2. **Qwen Code должен поддерживать** переменную `QWEN_SESSION_SUMMARY`
3. **Если автосохранение не работает** — используйте ручные команды

## 🆘 Troubleshooting

### Автосохранение не работает

**Проверьте:**
1. Существует ли файл `.qwen/extensions/superpowers/hooks/session-end`
2. Имеет ли он права на выполнение (Linux/Mac): `chmod +x session-end`
3. Прописан ли хук в `hooks.json`

**Альтернатива:**
Используйте ручные команды `/сохранить`

---

### Файлы не обновляются

**Проверьте:**
1. Путь к проекту в хуке: `PROJECT_ROOT="D:/GitHub/maribulka"`
2. Существует ли директория `memory/`
3. Права на запись в файлы
